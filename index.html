Mes Projets

Refonte d'un logiciel existant

J'ai repris un logiciel développé initialement en Visual Basic destiné à une utilisation desktop sans base de données partagée. Je suis partie de zéro en reprenant certaines fonctionnalités de l'ancien logiciel et en développant une toute nouvelle application Web en C# avec ASP.NET Framework. Dans ce projet, j'ai ajouté une base de données partagée SQL Server entièrement optimisée permettant aux utilisateurs d'avoir une interface Web rapide pour ajouter, modifier ou supprimer des données. J'ai également développé une authentification 2FA via adresse mail vérifiée et une interface complète de gestion du logiciel pour les administrateurs. À la suite de ce projet, j'ai formé les utilisateurs via une conduite du changement afin de les familiariser avec cette nouvelle interface. Le logiciel est entièrement Web 2.0 et responsive à l'aide du framework Bootstrap pour diriger une partie du style du site. Ce projet a été développé dans le cadre de mon alternance dans l'entreprise FRUEHAUF.

Vecto

Afin de respecter les nouvelles normes européennes en terme d'émission de CO₂ liée à la production de remorques. J'ai développé une application console entièrement automatisée permettant le calcul des émissions de CO₂ et le respect des poids et hauteurs réglementaires imposés par l'Union européenne. Cette application console a été entièrement développée en C# et se déclenche à chaque fois qu'un nouveau véhicule est ajouté dans notre ERP. Cette application a été développée dans le cadre de mon alternance chez FRUEHAUF.

Python/Flask

J'ai développé une application Python/Flask dans le cadre de mes études. Cette application est entièrement scalable et chaque fonction Python est séparée dans un sous-fichier. Le déploiement de l'application se fait via des images Docker générées par OpenFaaS et envoyées dans le registre d'images GitLab. Un cluster Kubernetes hébergé sur des machines virtuelles s'occupe de télécharger les images du répertoire et de gérer les ressources nécessaires puis de déployer les fonctions Python sur la VM Worker. L'adresse de l'application est ensuite extériorisée afin d'autoriser l'accès aux utilisateurs du réseau local.
